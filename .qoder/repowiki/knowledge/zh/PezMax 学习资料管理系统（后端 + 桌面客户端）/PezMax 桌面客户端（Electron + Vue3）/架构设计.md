采用 electron-vite 三进程架构：
- `src/main/index.js` 为单入口主进程，集中注册所有 IPC 处理器（窗口控制、设置持久化、全局快捷键、文件/文件夹选择、流式直写下载、SQLite 下载记录、自动更新），并通过 `main-utils/apiRegistry.js` 预留按名称查找的通用 API 路由。
- `src/preload/index.js` 通过 contextBridge 暴露 `electronAPI` 白名单方法，仅允许渲染进程以 `ipcRenderer.invoke('call-api', name, ...args)` 形式调用主进程能力，配合 `contextIsolation: true` / `nodeIntegration: false` 实现安全隔离。
- `src/renderer` 是标准 Vue3 + Element Plus + Pinia + Vue Router SPA，按业务域组织：`views/{datum,tool,home,...}` 页面、`components/*` 可复用组件、`api/{datum,monitor,system,tool}` 对应后端模块的 HTTP 封装、`store/modules` 状态切片、`directive` 权限指令、`utils/request.js` 统一 axios 请求层。
- 构建与打包由 `electron.vite.config.mjs` 驱动（renderer root=`src/renderer`，输出到 `out/renderer`；main/preload 分别走 alias `@main`/`@preload`），`electron-builder.yml` 配置 NSIS/DMG/AppImage 产物及 GitHub Releases generic 发布源，`dev-app-update.yml` 指定开发期更新源。
- 依赖方向单向：renderer → preload (IPC) → main (Node/fs/dialog/net/sql.js)，主进程不反向引用渲染代码。
