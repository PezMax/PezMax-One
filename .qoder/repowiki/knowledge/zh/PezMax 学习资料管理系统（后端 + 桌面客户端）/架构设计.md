仓库采用「后端多模块 + 独立桌面客户端」的双端结构：
- PezMax-Backend 为 Maven 多模块聚合工程，`ruoyi-common`/`ruoyi-framework`/`ruoyi-system`/`ruoyi-quartz`/`ruoyi-generator` 提供通用能力，`ptmj-datum` 承载业务领域层，`ruoyi-admin` 作为启动入口暴露 Controller；`ruoyi-ui` 是内嵌的 Vue3 管理前端。
- PezMax-Desktop 是独立的 Electron-vite 应用，其 `renderer` 目录复制了与 `ruoyi-ui` 同构的页面/组件/API 调用树，通过主进程 `src/main/main-utils/apiRegistry.js` 将本地 SQLite 数据库与远端 REST API 桥接给渲染进程。
- 两端共享同一套数据契约：后端 `ptmj-datum` 的 domain/dto/mapper/service 与前端 `ruoyi-ui/src/api/*`、`PezMax-Desktop/src/renderer/api/*` 一一对应，登录均走 `ruoyi-framework` 的 JWT 流程，权限由 `@Anonymous`/`@DataScope` 等注解在网关侧拦截。
- 部署上通过根级 `compose.yaml` 与 `Dockerfile` 统一编排后端容器，桌面端通过 `electron-builder.yml` 打包分发。
