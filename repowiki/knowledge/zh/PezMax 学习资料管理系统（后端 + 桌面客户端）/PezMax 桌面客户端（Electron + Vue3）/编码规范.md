- 主进程 IPC 统一使用 `ipcMain.handle` 返回 Promise 风格结果，错误路径一律包装 `{ success: false, message }` 对象而非抛异常，便于渲染端统一处理。
- 渲染端通过 `window.electronAPI.callApi(apiName, ...args)` 间接调用主进程函数，新增能力需在 `main-utils/apiRegistry.js` 中注册模块并在 `findApi` 中按名称解析，避免硬编码 IPC channel。
- 预加载脚本只通过 `contextBridge.exposeInMainWorld` 显式暴露白名单方法，禁止在渲染进程直接 require Node 模块，保持上下文隔离。
- 前端 API 层按后端模块划分目录（`src/renderer/api/{datum,monitor,system,tool}`），每个子目录一个 JS 文件导出该领域的所有接口，与 `views` 目录结构一一对应。
- 构建产物通过 `electron.vite.config.mjs` 中的 Rollup output.chunkFileNames/entryFileNames 模板统一加 hash，静态资源按 `static/[ext]/[name]-[hash].[ext]` 分类输出。
