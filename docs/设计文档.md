## 1 · 设计理念

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **Content over chrome** | 内容即界面，导航元素最小化 |
| **筛选优于浏览** | 用搜索 + 筛选代替文件树，更快定位 |
| **线性预览** | 主从视图（Master-Detail），无 Tab 管理 |
| **空间连续性** | 所有状态变化用 Sokuou Engine 连续解析，无瞬态跳变 |
| **频率驱动导航** | 高频功能（浏览）默认可见，低频功能（设置）收纳 |

### 1.2 用户画像

| 用户 | 行为 | 核心需求 |
|------|------|---------|
| 学生（找资料） | 搜索试卷 → 预览 → 下载 | 快速定位、一键下载 |
| 教师（找资料） | 按学科/学校筛选 → 浏览 → 收藏 | 分类筛选、收藏管理 |
| 贡献者（上传） | 上传文件 → 管理上传记录 | 批量上传、进度可见 |
| 访客（浏览） | 浏览排行 → 查看热门 | 发现优质资源 |

### 2.2 汉堡菜单（侧边栏）

```
│    │                          │                      │
│ ⚙️ │                          │ ⚙️ 个人              │
```

**规格参数：**

| 属性 | 值 |
|------|-----|
| 折叠态宽度 | 48px |
| 展开态宽度 | 200px |

| 层级 | 职责 | 形式 |
|------|------|------|

## 4 · 交互设计

### 4.1 PDF 查看策略

| 场景 | 方案 | 优先级 |
|------|------|--------|
| 快速预览 | 后端渲染 PDF 第一页为 PNG，在预览区显示 | P0 |
| 完整查看 | 下载后调用系统默认 PDF 阅读器打开 | P0 |
| 应用内翻页查看 | 集成 PDFium/MuPDF 逐页渲染 | P2 |

### 4.2 下载流程

```
用户点击 📥 下载文件 →
  1. 调用 `rfd::FileDialog::new().set_file_name(...).save_file()` 选择保存位置
  2. 显示进度条（Progress + EaseOutCubic）
  3. `reqwest` 流式下载到本地文件
  4. 完成后显示 Toast 提示
  5. 可选：自动调用系统阅读器打开
```

### 4.3 收藏流程

```
用户点击 ⭐ 收藏 →
  1. 立即更新 UI 状态（乐观更新）
  2. 按钮图标切换（⭐ → ★），颜色过渡（Progress, 0.2s）
  3. 异步调用收藏 API
  4. 失败时回滚状态 + Toast 提示
```

### 4.4 搜索流程

```
用户输入关键词 →
  1. 300ms 防抖（避免频繁请求）
  2. 调用搜索 API（`search_files?keyword=...`）
  3. 结果网格更新，旧卡片淡出，新卡片淡入
  4. 筛选器 chips 保持当前状态
```

### 4.5 Toast 通知

- 右下角叠层显示
- 进场：从右侧滑入（Progress, 0.25s, EaseOutCubic）
- 停留：5 秒自动消失，或点击关闭
- 离场：向上滑出 + 淡出（Progress, 0.2s, EaseInCubic）
- 最多同时显示 3 条，超出时堆叠

## 5 · 动画规格

### 5.1 动画参数速查

| 场景 | 原语 | 参数 | 属性 |
|------|------|------|------|
| 侧边栏展开/折叠 | SpringAnim | response=0.5, damping=0.825 | 宽度 48↔200px |
| 侧边栏选中切换 | SpringAnim | response=0.3, damping=0.8 | 高亮位置滑动 |
| 页面切换 | SpringAnim | response=0.4, damping=0.8 | 内容区渐显 + 轻微上移 |
| 首页卡片入场 | SpringAnim | response=0.5, damping=0.825 | 依次弹入，100ms 延迟 |
| 结果卡片入场 | SpringAnim | response=0.4, damping=0.8 | 错位渐显，50ms 延迟 |
| 卡片悬停 | SpringAnim | response=0.3, damping=0.6 | 上浮 2px |
| 预览区展开/收起 | SpringAnim | response=0.4, damping=0.8 | 右侧滑入 |
| 筛选器 chip 切换 | Progress | 0.15s, Linear | 颜色过渡 |
| 数字滚动 | Progress | 0.6s, EaseOutCubic | 0 → 目标值 |
| 收藏按钮 | Progress | 0.2s, EaseOutCubic | 图标 + 颜色切换 |
| 下载进度条 | Progress | 随下载进度, EaseOutCubic | 进度条填满 |
| Toast 进场 | Progress | 0.25s, EaseOutCubic | 右侧滑入 + 淡入 |
| Toast 离场 | Progress | 0.2s, EaseInCubic | 向上滑出 + 淡出 |
| 列表项淡入 | Progress | 0.3s, EaseOutCubic | 80ms 错位延迟 |

### 5.2 主次动画分离原则

```
主要运动（位置、大小）→ SpringAnim（物理感）
次要属性（透明度、颜色）→ Progress + EaseOut（非物理感）
```

从同一个 progress 派生所有属性，不为同一元素的不同属性建独立动画。

### 5.3 性能守则

- 稳态时 `ctx.request_repaint()` 不调用（egui 只在输入事件时重绘）
- `progress < 0.01` 时跳过渲染
- 稳态时使用精确值，不经过 progress 派生
- 每帧 `update(dt)` 只做数值计算，不做 I/O

## 6 · 技术架构

### 6.1 文件结构

```

├── app.rs                   # PezMaxApp 状态 + 路由 + eframe::App impl
```

### 6.3 数据流

```
用户操作 → PezMaxApp 状态变更 → Sokuou Engine update(dt)
  → 渲染函数读取状态 + 动画值 → egui 绘制
```

所有页面渲染函数**只读** `app` 状态，不修改。状态修改通过 `app` 的方法进行。

## 附录：对比总结

| 维度 | Electron 版 | egui 新设计 |
|------|-----------|-------------|
| 导航 | 7 个平级按钮，56px 固定 | 4 个分组，可折叠汉堡菜单 48↔200px |

| 下载 | 在 PDF 工具栏里 | 预览区底部显眼按钮 |
| 交互反馈 | 无（浏览器原生） | Sokuou Engine 弹簧动画 |
| 布局 | 三栏（ActivityBar/SidePanel/Editor） | 侧边栏 + 内容区 |
| UI 框架 | Vue 3 + Element Plus | egui + Metro Design |
