# PezMax 后端接口列表

> 用途：供前端（Electron + Vue3 + Element Plus）重置时对照。后端为基于 **RuoYi** 框架的 Spring Boot 多模块项目。
> 整理时间：2026-07-15　整理范围：`ruoyi-admin` / `ruoyi-generator` / `ruoyi-quartz` 下全部 34 个 Controller。

---

## 0. 关键提示：管理端 vs 桌面端（client / admin 两套入口）

本项目前端通过 `VITE_AUTH_ENTRY_MODE` 区分 **admin（管理端）** 与 **client（桌面端）** 两套入口，后端对应两套独立的鉴权与业务接口：

| 场景 | 管理端（admin） | 桌面端（client） |
|------|----------------|-----------------|
| 登录 | `POST /login` | `POST /datum/user/login` |
| 获取当前用户信息 | `GET /getInfo` | `GET /datum/user/getInfo` |
| 获取路由/菜单 | `GET /getRouters` | （桌面端路由在前端写死，不走后端） |
| 验证码 | `GET /captchaImage` | `GET /datum/user/captchaImage` |
| 注册 | `POST /register`（受配置开关控制） | `POST /datum/user/register`（自带 3 个密保问题） |
| 个人中心 | `/system/user/profile/*` | `/datum/desktop/user/profile/*` |
| 业务数据 | `system/*`、`datum/*`（管理端 CRUD + 审核） | `datum/desktop/*`、`datum/user/*`（只读 + 收藏/下载/举报） |

**业务实体双接口**：书签收藏、试卷收藏、试卷下载在管理端走 `/datum/bookmark/favorite`、`/datum/favorite`、`/datum/download`，在桌面端走 `/datum/desktop/bookmark/favorite`、`/datum/desktop/favorite`、`/datum/desktop/download`。重置前端时务必按入口分别对接。

## 1. 接口调用约定（前端 axios）

| 项 | 值 | 说明 |
|----|----|------|
| baseURL | `import.meta.env.VITE_APP_BASE_API`（开发 = `/dev-api`） | Electron 主进程将 `/dev-api` 代理到后端 `VITE_APP_TARGET_URL`（如 `http://154.8.139.48:8080`） |
| 超时 | 10000ms | |
| Content-Type | `application/json;charset=utf-8`（FormData 时自动移除） | |
| 鉴权头 | `Authorization: Bearer <token>` | 除标记「匿名」的接口外，所有接口需携带；token 来自登录返回 |
| 防重复提交 | POST/PUT 默认开启，1s 内同 url+data 拦截 | 可用 `headers: { repeatSubmit: false }` 关闭 |
| 不带 token | `headers: { isToken: false }` | |

### 响应统一格式

- **普通接口** `AjaxResult`：
  ```json
  { "code": 200, "msg": "操作成功", "data": {} , ...其它扩展字段 }
  ```
  `code=200` 成功；`401` 会话过期（前端弹重新登录）；`500` 业务错误；`601` 警告。
- **分页接口** `TableDataInfo`：
  ```json
  { "code": 200, "msg": "查询成功", "rows": [], "total": 100 }
  ```
  分页参数统一走 query：`pageNum`（默认 1）、`pageSize`（默认 10）、`orderByColumn`、`isAsc`。
- **导出/下载接口**：`responseType: 'blob'`，前端用 `download(url, params, filename)` 封装（POST + form-urlencoded）。
- **文件上传**：`multipart/form-data`，字段名通常为 `file`。

### 鉴权说明列含义
- **需登录**：无 `@Anonymous` 注解，请求须带 Bearer token。
- **匿名**：标注 `@Anonymous`，可不带 token 调用。
- **需权限**：带 `@PreAuthorize("@ss.hasPermi('xxx:xxx:xxx')")`，除登录外还需用户拥有该权限标识（RuoYi 权限体系）。
